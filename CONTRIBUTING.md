# Contributing to Cuentify

Thank you for considering contributing to **Cuentify**, a next-generation storytelling platform! We welcome contributions from everyone. Please follow the guidelines below to ensure a smooth collaboration.

## How to Contribute

1. **Fork the Repository**
   - Click the Fork button on the [Cuentify GitHub page](https://github.com/Olsoweirodev/Cuentify) to create your own copy.

2. **Clone Your Fork**
   `ash
   git clone https://github.com/YOUR-USERNAME/Cuentify.git
   cd Cuentify
   `ash

3. **Create a Branch**
   - Create a new branch for your changes:
     `ash
     git checkout -b feature/your-feature-name
     `ash

4. **Make Changes**
   - Make your changes in the codebase. Ensure your code follows the project's coding standards.
   - Test your changes locally:
     `ash
     npm run dev
     `ash

5. **Commit Your Changes**
   - Commit your changes with a descriptive message:
     `ash
     git add .
     git commit -m Add your descriptive message here`n     `ash

6. **Push to Your Fork**
   - Push your changes to your forked repository:
     `ash
     git push origin feature/your-feature-name
     `ash

7. **Create a Pull Request**
   - Go to the [Cuentify GitHub page](https://github.com/Olsoweirodev/Cuentify), and you’ll see an option to create a pull request from your branch.
   - Provide a clear description of your changes in the pull request.

## Code of Conduct

- Be respectful and inclusive.
- Avoid any form of harassment or discrimination.
- Provide constructive feedback.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on the [Cuentify GitHub Issues page](https://github.com/Olsoweirodev/Cuentify/issues).
